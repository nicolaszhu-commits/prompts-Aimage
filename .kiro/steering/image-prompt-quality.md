# 配图提示词质量把控规则

> 适用范围：本规则约束「Markdown 智能配图提示词提取流水线」生成的全部 `Image_Prompt`（生图提示词）与 `Prompt_Document`（配图提示词文档）。技能 `/生成配图提示词` 在执行时必须显式加载并严格遵循本文件；处理 `output/**`、`articles/**` 下的文件时本规则亦作为兜底自动加载。
>
> 关联需求：5.1、5.2、5.3、5.4（同时为 3.3、3.4、4.4 的三字段约束提供质量基线）。

生成提示词的唯一目标，是让用户能把每一条描述**原样复制**到第三方生图平台，得到忠于原文意图的配图。因此所有提示词必须同时满足以下四条规则。任何一条不满足，都视为不合格产出，必须修正后再写入 `output/`。

## 规则一：忠实性 —— 提示词高度忠于原文（需求 5.2）

- 每一条 `Image_Prompt` 的「画面 / 图表描述」都必须**高度忠于原文信息**，准确反映其所对应段落或小标题的核心含义、主题与语气。
- 描述中出现的主体、关系、数据、结论、场景要素，都应能在原文中找到直接依据。
- 在保留原文本意的前提下，可以把文字信息转译为画面语言（构图、元素、图表形态），但**不得改变原文要表达的事实与观点**。
- 数据密集型段落（`Data_Rich_Segment`）的描述应忠实呈现原文的量化关系（如对比、趋势、占比、分类）；纯逻辑段落（`Logic_Segment`）的描述应忠实呈现原文的概念、流程或逻辑关系。

## 规则二：不臆造 —— 仅依据原文已有信息（需求 5.3）

- **只能依据原文已经包含的信息**生成提示词，**严禁引入原文中不存在的内容、数据、结论或细节**。
- 具体禁止项：
  - 不得编造原文未给出的具体数字、百分比、年份、统计口径或来源。
  - 不得添加原文未提及的人物、机构、产品、地点、事件。
  - 不得推导或补全原文未明确表达的结论、因果或预测。
  - 不得用「常识」「行业惯例」替原文补充其没有写的内容。
- 当原文信息不足以支撑一条完整画面描述时，应**如实收敛**：只描述原文确有的要素，宁可描述简洁，也不臆造填充。
- 若某段落（或全文）内容为空或缺乏可提取结构，应报告「无可生成的提示词 / 内容不足」，而不是凭空生成（与空文章处理一致）。

## 规则三：可复制性 —— 结构清晰、字段齐全、易于复制（需求 5.4）

- 输出的 `Prompt_Document` 必须**结构清晰、排版易读**，让用户能快速定位并**独立复制任意单条提示词**前往第三方平台使用。
- 每条提示词的「画面 / 图表描述」应是**自包含**的：脱离上下文也能被生图平台理解，无需用户再做拼接或补充说明。
- 文档须遵循约定模板：
  - 文档头部声明：源文章、统一风格应用状态（已应用 / Style_Guide 缺失）、提示词总数。
  - 每条提示词以独立小节呈现，按原文行文顺序编号排列。
  - 各小节之间使用清晰的分隔，便于逐条复制。
- 描述文本应避免使用仅在本文档内部有意义的引用（如「见上文」「同前段」），以免复制后语义丢失。

## 规则四：三字段规范 —— 每条提示词必须齐全（需求 3.3、4.4）

每一条 `Image_Prompt` 都**必须且仅包含**以下三项内容，三项均不得为空：

1. **对应段落 / 小标题**：标明该提示词来源于原文的哪个段落或小标题，承载于条目标题位置。
2. **图片类型**：标明配图类别与具体形态。
   - `Data_Rich_Segment` → 「数据可视化」类（如：数据图表、信息图表、量化对比图）。
   - `Logic_Segment` → 「概念表达」类（如：逻辑对比图、概念思维导图、场景插画）。
3. **画面 / 图表描述**：一段可直接复制到第三方生图平台、忠于原文且自包含的具体描述。

- 三字段缺一不可；任何缺字段、空字段或字段错位的条目都不合格。
- 「图片类型」的类别必须与来源段落的分类一致（数据段落配数据可视化，逻辑段落配概念表达），不得错配。

## 质量自检清单（写入 `output/` 前必须逐条确认）

- [ ] 忠实性：每条描述都能在原文找到依据，未改变原文事实与观点。
- [ ] 不臆造：无任何原文之外的数字、结论、主体或细节；信息不足时如实收敛。
- [ ] 可复制性：文档结构清晰、条目独立、描述自包含，可逐条直接复制。
- [ ] 三字段：每条均含「对应段落/小标题、图片类型、画面/图表描述」且非空，类型与分类一致。
